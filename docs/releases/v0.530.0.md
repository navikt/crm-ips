# Release notes v0.530.0

## Ny funksjonalitet

### Søk på fødselsnummer

Jobbspesialister som får oppgitt fødselsnummer fra Nav og helse kan nå søke på fødselsnummer for å finne riktig person i Jobbsporet. Dette løser problemet med finne tilbake til bestemte deltakere man har fødselsnummeret til, uten å skrive inn navnet.

### Recruitment Program (beta)

Videre utvikling av Recruitment Program-funksjonaliteten:

- Flow oppdatert med startdato og beslutningslogikk
- Ny "Update Recruitment Program"-flyt for å oppdatere løpende program
- Tilgjengelig for beta-testere via ny custom permission `IPS Beta Feature Access`

### Oppdaterte merkelapper og statusveiledere for Utvidet oppfølging

Merkelapper, beskrivelser og statusveiledere for "Utvidet oppfølging" er oppdatert i tråd med JOBBSPORET-645. Blant annet er hurtighandlingene for møte med arbeidsgiver og nytt møte i UO oppdatert.

## Feilrettinger

### Fikset etiketter for Oppstartsdato og Henvist dato

Feilaktige etiketter for feltene `Oppstartsdato` og `Henvist dato` på Work_Trail-objektet er rettet.

### Forbedret tekst i hjelpeboble for deltakers mobilnummer

Utdatert og misvisende tekst i hjelpeboblen for feltet "Deltakers mobilnummer" er fjernet.

### Forbedret tilgangsstyring

Flyten "IPS – Admin Give User Access" er oppdatert med mer robust profilhenting og manglende profil i tildelingen er lagt til.

## Tekniske endringer

- SOQL-spørringer er oppdatert til å bruke positive uttrykk (erstattet `NOT IN` og `!=` med `IN`)
- `startsWith`-attributt er fjernet fra relevante komponenter
- Ugyldige `uiBehavior`-verdier i quickAction-filer er rettet
- Ubrukt workflow `push_deploy-to-dev1.yml` er slettet
- GitHub Actions-workflows har fått eksplisitte `permissions`
